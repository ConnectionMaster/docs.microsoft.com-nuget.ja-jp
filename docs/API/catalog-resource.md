<span data-ttu-id="ad764-418">クライアントを格納する必要があります、カタログは、時間によってインデックス設定される追加専用のデータ構造であるため、**カーソル**まで、どの時点でクライアントを表すが処理されるカタログ アイテムがローカルにします。</span><span class="sxs-lookup"><span data-stu-id="ad764-418">Because the catalog is an append-only data structure indexed by time, the client should store a **cursor** locally, representing up to what point in time the client has processed catalog items.</span></span> <span data-ttu-id="ad764-419">このカーソル値必要がありますしない生成されること、クライアントのコンピューター クロックを使用してに注意してください。</span><span class="sxs-lookup"><span data-stu-id="ad764-419">Note that this cursor value should never be generated using the client's machine clock.</span></span> <span data-ttu-id="ad764-420">カタログ オブジェクトの元の値となる代わりに、`commitTimestamp`値。</span><span class="sxs-lookup"><span data-stu-id="ad764-420">Instead, the value should come from a catalog object's `commitTimestamp` value.</span></span>

<span data-ttu-id="ad764-421">たびに、クライアントがパッケージ ソースに新しいイベントを処理しようとすると、その必要がありますカタログのクエリのみ、コミット タイムスタンプのすべてのカタログ アイテムのストアド カーソルより大きい。</span><span class="sxs-lookup"><span data-stu-id="ad764-421">Every time the client wants to process new events on the package source, it need only query the catalog for all catalog items with a commit timestamp greater than its stored cursor.</span></span> <span data-ttu-id="ad764-422">クライアントは、すべての新しいカタログ アイテムを適切に処理を後にだけその新しいカーソル値として処理されたカタログ アイテムの最新のコミット タイムスタンプを記録します。</span><span class="sxs-lookup"><span data-stu-id="ad764-422">After the client successfully processes all new catalog items, it records the latest commit timestamp of catalog items just processed as its new cursor value.</span></span>

<span data-ttu-id="ad764-423">このアプローチを使用して、クライアントはパッケージ ソースで発生したすべてのパッケージ イベントを受信しないことを確認できます。</span><span class="sxs-lookup"><span data-stu-id="ad764-423">Using this approach, the client can be sure to never miss any package events that occurred on the package source.</span></span>
<span data-ttu-id="ad764-424">さらに、クライアントは、カーソルの記録されたコミットのタイムスタンプより前の古いイベントを再処理することはありませんが。</span><span class="sxs-lookup"><span data-stu-id="ad764-424">Additionally, the client never has to reprocess old events prior to the cursor's recorded commit timestamp.</span></span>

<span data-ttu-id="ad764-425">このカーソルの強力な概念は、nuget.org のバック グラウンド ジョブの多くのために使用され、V3 API 自体を最新に保つために使用されます。</span><span class="sxs-lookup"><span data-stu-id="ad764-425">This powerful concept of cursors is used for many of nuget.org background jobs and is used to keep the V3 API itself up-to-date.</span></span>
