### <a name="nu1601"></a>NU1601

| --- | --- |
| **問題点** | 直接プロジェクトの依存関係が、指定したプロジェクトより新しいバージョンに高くなります。 |
| **メッセージの例** | *指定された依存関係が NuGet.Versioning (> = 3.5.0) NuGet.Versioning 4.0.0 で終了しましたが、します。* |
| **解決方法** | プロジェクト内の依存関係を適切なバージョンに更新します。 |

### <a name="nu1602"></a>NU1602

| | |
| --- | --- |
| **問題点** | パッケージの依存関係には、下限値がありません。 これは、検索への復元を許可しない、*最適一致*です。 各復元は、使用できる、古いバージョンを検索しようとするは下位変化します。 これは、その復元が毎回ユーザー パッケージ フォルダーに既に存在するパッケージを使用する代わりにすべてのソースを確認するオンラインになることを意味します。 |
| **メッセージの例** | *NuGet.Packaging 4.0.0 では、依存 NuGet.Versioning (> 3.5.0) のため、下限は提供されません。3.6.0 のおおよその最適な一致が解決されました。* |
| **解決方法** | これは通常、パッケージ作成エラーです。 この問題を解決するのには、パッケージの作成者に問い合わせてください。 |

### <a name="nu1603"></a>NU1603

| | |
| --- | --- |
| **問題点** | パッケージの依存関係は、バージョンが見つかりませんでしたを指定します。 通常、パッケージ ソースでは、予想される下限バージョンは含まれません。 以降のバージョンは、に対して、パッケージが作成されたものと異なるを代わりに、使用されました。<br/><br/>つまり、復元が見つかりませんでした、*最適一致*です。 各復元は、使用できる、古いバージョンを検索しようとするは下位変化します。 これは、その復元が毎回ユーザー パッケージ フォルダーに既に存在するパッケージを使用する代わりにすべてのソースを確認するオンラインになることを意味します。 |
| **メッセージの例** | NuGet.Packaging 4.0.0 NuGet.Versioning によって異なります (> = 4.0.0) が、4.0.0 が見つかりませんでした。 5.0.0 以降のおおよその最適な一致が解決されました。 |
| **解決方法** | 期待どおりパッケージがリリースされていない場合、パッケージの作成エラーでもかまいません。 この問題を解決するのには、パッケージの作成者に問い合わせてください。 パッケージがリリースされている場合、上にある使用可能なパッケージ ソースを使用しているを調べます。 プライベートのソースを使用する場合は、そのフィードのパッケージを更新する必要があります。 |

### <a name="nu1604"></a>NU1604
