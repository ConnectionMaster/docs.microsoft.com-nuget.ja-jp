### <a name="nu1601"></a>NU1601

| | |
| --- | --- |
| **問題点** | パッケージの依存関係には、下限値がありません。 これは、検索への復元を許可しない、*最適一致*です。 各復元は、使用できる、古いバージョンを検索しようとするは下位変化します。 これは、その復元が毎回ユーザー パッケージ フォルダーに既に存在するパッケージを使用する代わりにすべてのソースを確認するオンラインになることを意味します。 |
| **一般的な原因** | これは通常、パッケージ作成エラーです。 |
| **メッセージの例** | *NuGet.Packaging 4.0.0 では、依存 NuGet.Versioning (> 3.5.0) のため、下限は提供されません。3.6.0 のおおよその最適な一致が解決されました。* |

### <a name="nu1603"></a>NU1603

| | |
| --- | --- |
| **問題点** | パッケージの依存関係は、バージョンが見つかりませんでしたを指定します。 以降のバージョンは、に対して、パッケージが作成されたものと異なるを代わりに、使用されました。<br/><br/>つまり、復元が見つかりませんでした、*最適一致*です。 各復元は、使用できる、古いバージョンを検索しようとするは下位変化します。 これは、その復元が毎回ユーザー パッケージ フォルダーに既に存在するパッケージを使用する代わりにすべてのソースを確認するオンラインになることを意味します。 |
| **一般的な原因** | パッケージ ソースでは、予想される下限バージョンは含まれません。 期待どおりパッケージがリリースされていない場合、パッケージの作成エラーでもかまいません。 |
| **メッセージの例** | NuGet.Packaging 4.0.0 NuGet.Versioning によって異なります (> = 4.0.0) が、4.0.0 が見つかりませんでした。 5.0.0 以降のおおよその最適な一致が解決されました。 |

### <a name="nu1604"></a>NU1604

| | |
| --- | --- |

| **一般的な原因** | プロジェクトの*PackageReference* *バージョン*属性は、下限の境界を含むように更新する必要があります。 |
| **メッセージの例** | *プロジェクトの依存関係 NuGet.Versioning (< = 9.0.0) doe が包括的の下限値が含まれていません。結果の整合性の復元に依存関係のバージョンに下限の境界を含めます。* |

| | |
| --- | --- |

| **一般的な原因** | パッケージを解決するときに最も近いが優先されます。 グラフ内の最も近いパッケージには、離れた場所にあるパッケージがオーバーライドされている可能性があります。 |
